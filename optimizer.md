# Barra 优化器笔记

这份笔记整理最近几轮关于 Barra 风险模型、组合优化目标、交易成本、无闭式解算法和收敛证明的讨论。

## 1. Barra 在优化器里的位置

Barra 风险模型通常写成：

```text
Sigma = X F X' + D
```

其中：

- `Sigma` 是股票收益协方差矩阵
- `X` 是股票因子暴露矩阵
- `F` 是因子收益协方差矩阵
- `D` 是特质风险矩阵，通常近似为对角矩阵

Barra 本身主要不是预测收益，而是预测和解释风险。

优化器通常把 Barra 用在风险项里：

```text
risk(w) = w' Sigma w
```

利用因子结构可以改写成：

```text
w' Sigma w = (X'w)' F (X'w) + w'Dw
```

直觉：

- `X'w` 是组合因子暴露
- `(X'w)'F(X'w)` 是组合因子风险
- `w'Dw` 是组合特质风险

## 2. 组合方差为什么是 `w'Sigma w`

组合收益是个股收益的加权和：

```text
R_p = w1 R1 + w2 R2 + ... + wN RN = w'R
```

风险用方差衡量：

```text
Var(R_p)
```

根据方差展开公式：

```text
Var(R_p) = sum_i sum_j wi wj Cov(Ri, Rj)
```

而协方差矩阵 `Sigma` 的元素是：

```text
Sigma_ij = Cov(Ri, Rj)
```

所以：

```text
Var(R_p) = w' Sigma w
```

这表示所有股票两两协方差，按两个股票权重的乘积加权求和。

## 3. 优化器目标不是只让风险最低

组合优化通常不是单纯最小化风险，而是在收益、风险、成本和约束之间权衡。

典型目标：

```text
max_w alpha'w - lambda/2 * w'Sigma w - TC(w)
```

其中：

- `alpha'w` 是组合预期收益
- `w'Sigma w` 是组合方差
- `TC(w)` 是交易成本
- `lambda` 是风险厌恶系数

`lambda` 越大，优化器越保守；`lambda` 越小，优化器越愿意为了收益承担风险。

## 4. 风险惩罚怎么衡量

风险惩罚常写成：

```text
risk_penalty = lambda/2 * w'Sigma w
```

它表示为了获得 alpha，需要付出的风险代价。

如果是相对基准组合，令：

```text
h = w - b
```

其中 `b` 是基准权重，主动风险是：

```text
h' Sigma h
```

优化目标常写成：

```text
max_w alpha'h - lambda/2 * h'Sigma h - TC(w)
```

这里的风险是 tracking error 的平方。

## 5. 收益减方差不是归一化

目标函数：

```text
alpha'w - lambda/2 * w'Sigma w
```

不是严格意义上的归一化，而是均值-方差效用函数。

量纲上：

- `alpha'w` 是收益率
- `w'Sigma w` 是收益率平方
- `lambda` 把方差换算成收益层面的惩罚

也可以不用风险惩罚形式，而改成硬约束：

```text
max_w alpha'w
subject to w'Sigma w <= risk_limit
```

或者：

```text
min_w w'Sigma w
subject to alpha'w >= target_return
```

## 6. 交易成本怎么放进去

令：

```text
w_old = 当前持仓权重
w = 目标持仓权重
u = w - w_old
```

线性交易成本：

```text
TC(w) = sum_i c_i |w_i - w_old_i|
```

二次冲击成本：

```text
TC(w) = sum_i q_i (w_i - w_old_i)^2
```

组合使用：

```text
TC(w) = sum_i c_i |w_i - w_old_i| + sum_i q_i (w_i - w_old_i)^2
```

含义：

- 换仓越大，成本越高
- 流动性越差，成本越高
- alpha 不足以覆盖成本时，优化器不应交易

绝对值成本可以用辅助变量线性化：

```text
u_i >= w_i - w_old_i
u_i >= -(w_i - w_old_i)
u_i >= 0
```

此时 `u_i` 就代表 `|w_i - w_old_i|`。

## 7. 常见约束

组合优化常见约束：

```text
sum(w) = 1
w_i >= 0
w_i <= upper_i
sum_i |w_i - w_old_i| <= turnover_limit
行业暴露在上下限内
风格暴露在上下限内
主动风险不超过上限
```

行业和风格约束可以写成：

```text
lower <= X'(w - b) <= upper
```

其中 `b` 是基准权重。

## 8. Alpha 和 Barra 因子的关系

Alpha 是收益预测信号，Barra 因子是风险解释变量。

两者可以重叠，但角色不同：

- Alpha: 想赚什么
- Barra: 要控制什么风险

如果某个风险因子历史收益长期为正，只说明它过去被市场奖励过，不等于未来一定值得投资。

更严谨的检验包括：

- 均值是否显著为正
- 样本外是否有效
- 分层收益是否单调
- 多空组合是否稳定
- 交易成本后是否仍然有效
- 是否只是吸收了其他遗漏因子

## 9. 闭式解和数值优化

闭式解英文是：

```text
closed-form solution
```

例如加权最小二乘有闭式解：

```text
f_hat = (X'WX)^(-1) X'Wr
```

但组合优化一旦加入：

- 持仓上下限
- 行业约束
- 风格约束
- 交易成本
- 换手约束
- tracking error 约束

通常就没有简单闭式解，需要数值优化。

## 10. 开发优化算法的思路

如果没有闭式解，先判断问题结构：

```text
目标函数是否凸
约束集合是否凸
是否是 QP / SOCP / LP
是否有绝对值等非光滑项
变量规模有多大
```

常见类型：

- 线性目标 + 线性约束: LP
- 二次目标 + 线性约束: QP
- 二次风险约束: SOCP
- 绝对值成本: 可用辅助变量或 proximal 方法

实务优先路线：

```text
1. 先用 cvxpy / OSQP / Gurobi / MOSEK 建模跑通
2. 确认可行性、约束、风险和成本都正确
3. 如果性能不够，再考虑自研算法
```

自研常见算法：

- 投影梯度法
- 坐标下降
- ADMM
- 牛顿法或拟牛顿法
- 内点法

## 11. 为什么凸性重要

如果问题是凸优化：

```text
局部最优 = 全局最优
```

例如最小化：

```text
1/2 w'Sigma w - alpha'w
```

如果 `Sigma` 半正定，则：

```text
1/2 w'Sigma w
```

是凸函数，线性项 `-alpha'w` 仍然保持凸性。

线性等式和线性不等式约束构成凸可行域。

因此这是凸优化问题。

如果 `Sigma` 正定，目标函数通常更稳定，并可能得到唯一解。

## 12. 没有闭式解时怎么证明收敛

收敛证明通常不是证明一步算出答案，而是证明迭代序列会越来越接近最优解，或目标函数值收敛到最优值。

常见证明链路：

```text
1. 证明最优解存在
2. 证明问题是凸的
3. 证明算法每一步让目标函数下降
4. 证明目标函数有下界
5. 得到目标函数值收敛
6. 证明极限点满足 KKT 条件
7. 对凸问题，KKT 条件推出全局最优
```

最优解存在的常见条件：

```text
目标函数连续
可行域非空
可行域闭且有界
```

投影梯度法的典型条件：

```text
目标函数凸
梯度 Lipschitz 连续
可行域闭凸
步长 eta <= 1/L
```

ADMM 的典型条件：

```text
目标函数闭、真、凸
约束是线性的
增广拉格朗日参数 rho > 0
鞍点存在
```

## 13. 实现检查清单

开发 Barra 优化器时，可以按这个顺序检查：

```text
1. 股票顺序是否一致
2. alpha、X、F、D、w_old、benchmark 是否对齐
3. F 是否半正定
4. D 的对角线是否非负
5. 风险计算是否使用 Barra 分解而不是巨大 Sigma
6. 目标函数符号是否正确
7. 权重约束是否满足
8. 行业和风格约束是否按主动权重计算
9. 交易成本是否按换仓量计算
10. 优化结果是否可解释
```

## 14. 一句话总结

Barra 优化器的核心是：

```text
用 alpha 表达收益观点，
用 XFX' + D 表达风险结构，
用交易成本约束换仓，
用凸优化算法在约束下找到风险调整后的最优组合。
```
